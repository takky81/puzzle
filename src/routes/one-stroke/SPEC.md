# 一筆書きパズル 仕様書

## 概要

グリッド上の通行可能な全マスを一度ずつ通り、始点に戻る1つの閉じたループ（ハミルトン閉路）を見つけるパズル。

## ルール

- グリッド上のマスを上下左右（4方向）に移動してつなぐ
- マス目の中心を通る線でつなぐ
- 通行可能な全てのマスを1回ずつ通る必要がある
- 始点と終点がつながり、1つの輪になればクリア
- 輪が複数に分かれるのはNG
- 線が交差するのはNG、線が接するのはOK
- 始点はユーザーが自由に選ぶ。連続していない線も引ける
- すでに線が引かれたマスを再度なぞると線を削除できる
- undo/redoは1ストローク単位で行う（1ストローク = 指を離すまでの一連の移動。方向転換しても継続）
- 線の削除も同様に1ストローク単位でundo/redoされる
- リセットで最初からやり直し

## ステージ

- ステージは動的に生成する
- 正解は必ず1通りのみ
- グリッドサイズは設定可能（デフォルト: 8x8）
- 難易度はグリッドサイズで調整
- グリッド内に通行不可の壁マスを配置し、解の一意性を確保する
- 壁マスの数はできるだけ少なくする

## ステージ生成アルゴリズム

- 答えのパターンがプレイヤーに推測されないようにする
- 以下のいずれか効果的な方を採用する:
  - **案A**: ランダムにハミルトン閉路を生成 → 壁マスを1つずつ追加して2つ目の解を潰し、一意性を確保する
  - **案B**: スパニングツリー法で初期解を生成 → ランダムに大きく変形してパターンを崩す → 壁マスを追加して一意性を検証
- 大きいグリッドで生成に時間がかかる場合はWeb Workerで非同期生成し「生成中...」を表示

## UI仕様

- **マス目**: グリッド表示。未訪問=グレー、現在地=強調色、通過済み=薄色
- **壁マス**: 通行不可。他のマスと区別できる色で表示
- **線**: マス目の中心を結ぶ線で表示。通過済み=強調色
- **アニメーション**: マス通過時に色が変わるアニメーション
- **進捗表示**: 「○/○ マス通過」
- **メッセージ**: ボードが隠れない固定位置に表示（またはトースト）
  - 全マス通過したが始点に戻れていない場合など、状況に応じたメッセージ
- **戻るボタン**: 1ストローク戻す
- **ギブアップボタン**: 正解のルートを表示する
- **リセットボタン**: ステージをやり直す
- **クリア表示**: 全マス通過+始点接続時に「クリア!」+ 新しいステージ生成

## 操作方法

- **スマホ**: ドラッグで連続してマスをなぞり線を引く
- **PC**: ドラッグで連続してマスをなぞり線を引く

## レイアウト

- パズル領域は画面幅に合わせて調整（max-width: 400px程度）
- 上部にグリッドサイズと進捗
- 下部に戻る・リセットボタン
