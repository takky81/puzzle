# ナンバーリンク 仕様書

## 概要

グリッド上に配置された番号ペアを線でつなぐパズル。全ペアを結ぶためには必然的に全マスを通過する設計のみを出題する。

## ルール

- n×n の矩形グリッド（壁マスなし）
- 1〜K の番号が各1ペア（2マス）ずつグリッドに初期配置されている
- プレイヤーは同じ番号同士を、隣接マス中心を結ぶ4方向（上下左右）の線でつなぐ
- 線の制約:
  - **交差不可**: 同一マスを異なる番号の線が通らない
  - **分岐不可**: 各マスに同一番号の線が通るのは高々2方向（＝1本の単純パス）
- **クリア条件**: 全ての番号ペアが線でつながったとき
- 出題する問題は **全ペアを結ぶと必然的に全マスが被覆される**ものに限定する
- **2×2 モノクロブロック禁止**（いわゆる「コの字型の遠回り」排除）
  - どの 2×2 の正方形も、同じ番号の線で4マスすべてが埋まらない
  - 異なる番号が同居する 2×2 は許容

## ステージ

- 正解は1通りのみ
- 壁マスは無し（常に n×n の矩形グリッド）
- サイズ: **4×4 〜 9×9**
  - 大きいサイズで生成が困難な場合、可能な範囲のサイズに限定する
- 番号ペア数 K:
  - サイズごとに **K を小さい順に試す**
  - 有効な問題（2×2モノクロ禁止 + 全マス被覆 + 一意解）を累積100問以上集める
  - 収集できた K 範囲の問題を全て保存
- 事前生成して静的ファイルとしてバンドル: `static/puzzles/numberlink/NxN.json`
- ゲーム開始時にランダム抽選

## 生成アルゴリズム

Rust で事前生成する（`scripts/generate_numberlink/`）。

1. n×n グリッド上で、K を 2 から順に増やしながら問題を探索
2. 各 K について、以下を満たす「端点ペア配置」を列挙・検査:
   - 全マスを被覆する K 本の単純パスに分割可能（= 全ペアを結ぶと全マス通る）
   - 2×2 モノクロブロックが存在しない
   - 一意解を持つ
3. 累積問題数が 100 に到達したら次のサイズへ
4. 探索には制約伝播付きバックトラック（一筆書き生成器と同種の手法）を流用

詳細なアルゴリズム（ハミルトンパス分解法 or 直接列挙）は実装時に決定する。

## データ形式

`static/puzzles/numberlink/NxN.json`:

```json
{
  "puzzles": [
    {
      "size": 5,
      "numbers": [
        {
          "id": 1,
          "positions": [
            [0, 0],
            [4, 4]
          ]
        },
        {
          "id": 2,
          "positions": [
            [0, 4],
            [2, 2]
          ]
        }
      ],
      "solution": [
        {
          "id": 1,
          "path": [
            [0, 0],
            [1, 0],
            [2, 0],
            [3, 0],
            [4, 0],
            [4, 1],
            [4, 2],
            [4, 3],
            [4, 4]
          ]
        },
        {
          "id": 2,
          "path": [
            [0, 4],
            [0, 3],
            [0, 2],
            [0, 1],
            [1, 1],
            [1, 2],
            [1, 3],
            [1, 4],
            [2, 4],
            [2, 3],
            [2, 2]
          ]
        }
      ]
    }
  ]
}
```

- `size`: グリッドの一辺の長さ
- `numbers[i].id`: 番号 ID（1始まり）
- `numbers[i].positions`: 番号が配置される2マス（行・列）
- `solution[i].path`: 該当 ID の正解パス（端点から端点までのマス列）

## UI仕様

### 表示

- **マス目**: n×n のグリッド表示
- **番号マス**: 番号 ID を数字で表示。ID ごとに色分け
- **線**: 隣接マス中心を結ぶ線で描画。該当番号の色で表示
- **被覆済みマス**: 線が通過済みのマスは該当番号の薄色で塗る
- **進捗表示**: 「○/○ ペア接続」

### 操作

- **スマホ/PC 共通**: ドラッグで線を引く
  - 番号マスまたは既存線の末端から開始
  - ドラッグ中のマスを連続してなぞると線が伸びる
  - 線は上下左右のみ（斜め不可）
  - 既に他の番号が通るマスに入ろうとしたら、その方向への延長を無視
- **線の削除**: 同じ線を末端から逆なぞりすると、なぞった分だけ削除
- **undo/redo**: 1ストローク単位（指を離すまでの一連の操作）
- **リセット**: 全線をクリア
- **ギブアップ**: 正解を表示

### クリア

- 全ペアが正しくつながったら「クリア!」を表示し、新しいステージを生成
- 全ペアがつながっても 2×2 モノクロや全マス未被覆があれば解ではない
  - ただし出題する問題は「全ペア接続 ⇒ 全マス被覆」を満たすため、ユーザー視点では「全ペア接続 = クリア」

### レイアウト

- モバイルファースト（max-width: 400px 程度）
- 上部にサイズ・進捗
- 下部にリセット・ギブアップ・戻る（undo）ボタン

## 色の方針

- 番号ごとに色分け。K の上限に合わせて識別しやすい色パレットを用意
- 被覆済みマスは番号色の薄色（背景色より濃いが、線より薄い）
- 共通UI（ヘッダー、ボタン）はアプリ全体のカラースキームを踏襲
